## Design Context

### Users
Lab operators (1-3 concurrent) managing a private distributed hash cracking cluster of 7 rigs. They are technical security professionals who value precision and efficiency. They interact with the dashboard to monitor agent health, orchestrate campaigns, track crack results, and manage resources. The interface is their command center — it should feel like a well-built instrument, not a consumer product.

### Brand Personality
**Precise, Powerful, Dark** — Technical authority with craftsmanship. HashHive is a serious security tool that respects its operator's expertise. Every element should feel intentional and engineered.

### Aesthetic Direction
- **Theme**: Dark only — Catppuccin Macchiato palette
  - Base: `#24273a` (background), `#1e2030` (surface/sidebar), `#363a4f` (overlay/borders)
  - Text: `#cad3f5` (primary), `#a5adcb` (secondary/muted), `#6e738d` (subtle)
  - Accent colors from Catppuccin Macchiato:
    - Primary accent: `#f5a97f` (peach) — warm, ties to "hive" branding
    - Success: `#a6da95` (green)
    - Warning: `#eed49f` (yellow)
    - Error: `#ed8796` (red)
    - Info: `#8aadf4` (blue)
    - Active/Running: `#91d7e3` (teal)
  - Surface hierarchy: base → mantle (`#1e2030`) → crust (`#181926`)
- **References**: Linear/Vercel (polish, animation, typography), Grafana/Datadog (data density, status-awareness), Hack The Box (security-flavored identity)
- **Anti-references**: Generic Bootstrap admin panels, bright consumer SaaS, overly playful UI, anything that looks like a template
- **Typography**: Inter or system-ui — clean, technical, excellent at small sizes for data-dense views
- **Motion**: Subtle, purposeful — fast transitions (150-200ms), no bouncing or playful animations

### Design Principles

1. **Data-forward**: The interface exists to surface information. Minimize chrome, maximize signal. Status and metrics should be immediately scannable.

2. **Precision over decoration**: Every visual element should earn its place. Prefer clean lines, consistent spacing, and semantic color over ornamental design. If it doesn't help the operator, remove it.

3. **Dark-native**: Not a light theme with colors inverted — designed from the ground up for dark backgrounds. Leverage Catppuccin Macchiato's carefully calibrated contrast ratios and pastels-on-dark aesthetic.

4. **Status at a glance**: Agents, campaigns, and tasks have lifecycles. Use color, shape, and position to communicate state instantly — the operator should never have to read a label to know if something is healthy.

5. **Crafted, not themed**: Feels like a purpose-built security tool, not a generic dashboard with a dark skin. Small details matter — monospace for hashes, semantic density for tables, intentional hierarchy in every view.
